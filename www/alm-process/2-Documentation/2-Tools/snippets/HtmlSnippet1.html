The M3X project includes some ANT tasks that are suited to being integrated in a build pipeline.

M3X to Binary

Probably the most important task that the M3X project, and by extension the xml format provides. This intends to be the easiest and least error prone way to convert source assets to JSR184 and JSR297 targeted assets.

To get this task working in your pipeline you need to follow these steps:

  1. Download the latest distribution of M3X. From it you will need the m3x.jar file.
  2. Place the m3x.jar file in your project library folder, or somewhere where your script can access it. The rest of this guide assumes that location is ${lib.dir}
  3. Place the m3x.jar file in your project library folder, or somewhere where your script can access it. The rest of this guide assumes that location is ${lib.dir}
  4. Also copy the JAXB implementation jar file, jaxb-impl.jar, from the distribution. Please note the license file that comes with it and ensure you agree to it before use. From the jaxb folder you will need the jaxb-impl.jar only. Hopefully this can be corrected in the future but due to the dynamic loading internally of the JAXB context (beyond our control) the taskdef below does not resolve all the references to JAXB. Put the jaxb-impl.jar in the same lib folder as m3x.jar, yet again assumed to be ${lib.dir}
  5. Add the following taskdef to your script:
    <taskdef name="m3x_to_m3g"
         classname="m3x.translation.m3g.XmlToBinaryTask">
  <classpath>
    <pathelement location="${lib.dir}/m3x.jar"/>
  </classpath>
</taskdef>
  6. Use the task in your script at the appropriate place (in this example the files are assumed to be in the ${res.dir} location): 
    <m3x_to_m3g>
  <classpath>
    <pathelement location="${lib.dir}/m3x.jar"/>
    <pathelement location="${lib.dir}/jaxb-impl.jar"/>
  </classpath>
  <fileset dir="${res.dir}">
    <include name="**/*.m3x"/>
  </fileset>
  <mapper type="glob" from="*.m3x" to="*.m3g"/>
</m3x_to_m3g>
  7. That's it. The task uses up-to-date comparisons so you should be able to run the task often without slowing down your build.

Blender export script

Blender is an amazing open source 3D DCC tool. Whilst the interface my be daunting to many newcomers the wide array of powerful features has ensured it has a vibrant community. Supporting Blender with M3X is a high priority and as such we have created an M3X exporter for it.

Currently the exporter is in an incomplete state, but static geometry, materials and lighting work quite well. For a more mature exporter please see Gerhard VÃ¶lkl's excellent m3g_export script.

To get the exporter installed you need to follow these steps

  1. Work out the folder of your blender installation that is the correct place to put python scripts.
    • Mac and Linux: this is usually ~/.blender/scripts
    • Windows XP: probably C:\Documents and Settings\<Username>\Application Data\Blender Foundation\Blender\.blender\scripts
    • Windows Vista: probably C:\Users\<Username>\AppData\Roaming\Blender Foundation\Blender\.blender\scripts
    See the Blender manual reference page on extensions for more information.
  2. Copy the blender/m3x_export.py script from the M3X distribution to the Blender scripts location.
  3. Export to your hearts content.
  4. Report any bugs to the Issue Tracker.
